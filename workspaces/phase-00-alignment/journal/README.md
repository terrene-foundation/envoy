# Phase 00 Journal

Decision records for Phase 00 work. Numbered in order.

## Voice

All journal entries use Foundation voice:

- Record the decision, the rationale in Envoy-positive terms, and the consequences.
- Do not frame alternatives adversarially. If a decision was between two external products or approaches, record it as "we chose X because X satisfies Envoy constraint Y" — not as a critique of the alternative.
- Keep entries factual and citeable. If a claim depends on external research, cite the source.

If a Phase 00 decision proves load-bearing for Envoy's architecture, promote it to an ADR in `DECISIONS.md`.

## Format

```
# 0001 — <short decision title>

**Date:** YYYY-MM-DD
**Status:** Proposed | Accepted | Superseded by #NNNN

## Context

<what situation triggered this decision>

## Decision

<the decision itself>

## Rationale

<why this decision satisfies Envoy's constraints>

## Consequences

<what changes as a result, and what follow-up is needed>
```
