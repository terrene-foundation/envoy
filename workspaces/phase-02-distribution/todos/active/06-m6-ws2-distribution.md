# M6 — WS-2 Distribution (Track B — release-gated)

**Milestone value + Track-B release-gating.** WS-2 delivers the Phase-02 _headline_ deliverable the user's roadmap names first — the **single static binary** (`ROADMAP.md:74` "Goal: single static binary"; `briefs/00-phase-02-scope.md` §WS-2; EC-02.1/2/9/10 per `01-analysis/00-objectives-to-deliverables.md:28-37`). It packages the WS-1 binary into 5 (6) OS targets under 50 MB, stands up the N=3 signed-mirror + reproducible-build supply-chain spine, and lands the first-run onboarding + ADR-0006 offline degraded-mode path. **This milestone is Track B (`02-plans/01-architecture.md` § Legal-gate-aware build sequence):** _building, testing, and signing-with-internal-keys the entire pipeline under a CODENAME is buildable NOW_ — autonomous-execution work, fully parallel with the (calendar-bound, NOT 10x-multipliable) legal track. **Only three RELEASE-time actions are legally gated** (ADR-0009 `:303-313`): (1) publishing under the trademarked name on the public channels, (2) attaching the composite LICENSE, (3) cross-border crypto _redistribution_. **EC-02.1 binary BUILD is explicitly NOT export-controlled** — only redistribution is (EAR 742.15(b)(1), notification-email-satisfiable; `01-analysis/00-objectives-to-deliverables.md:28`; `02-ws2-distribution.md:9,168`). Each todo below tags which acceptance gates run **NOW** (codename, internal keys) vs **AT RELEASE** (legal-gated). The single HIGH-risk cell — musl-static + embedded CPython — is isolated in its own shard (S15m) with an Alpine import-smoke gate and a defer-to-gnu-only fallback valve so it cannot poison the size-gate or the rest of the milestone.

**Legal-gate boundary (carry into every shard below).** Buildable NOW under codename + internal signing keys: 5(6)-target cross-compile, <50 MB size gate, musl import-smoke, N=3 mirror-verify logic + ≥2/3 quorum, reproducible-build determinism, key-rotation + revoked-key refusal, cert-pinning install channel, first-run/upgrade/rollback/uninstall, install-to-first-value measurement (`02-ws2-distribution.md:144-160`). RELEASE-gated (cannot ship until the open ADR-0009 legal item closes): public-name publish (trademark `:312`), composite-LICENSE attach (`:307`), cross-border crypto redistribution (export-control `:311`). Wire each release gate as a release-time CI check that hard-fails if the legal artifact is absent (`DECISIONS.md:322` precedent), NOT a human checklist (`02-ws2-distribution.md:170-177`).

## S15 — 5(6) build targets + <50MB embed + size CI gate (codename)

- **Type:** Build
- **Value-anchor:** `ROADMAP.md:74` headline "single static binary" + `briefs/00-phase-02-scope.md` §WS-2 "Binary builds on 5 targets … **<50 MB** with embedded interpreter" (EC-02.1 + EC-02.2, `01-analysis/00-objectives-to-deliverables.md:28-29`). This is the literal artifact the user's sovereignty pitch rests on — one self-contained binary, no system Python, no npm supply chain (`DECISIONS.md:73`).
- **Implements:** `specs/distribution.md` §Phase 02 distribution (`:19-25`), §Binary size constraint (`:78-84`); ADR-0001 cross-compilation matrix (`DECISIONS.md:98`) + embedding cost pre-acceptance (`:97` "+30–40 MB. Accept").
- **Depends:** **S2a** (the rs-bindings adapter must be wired before its binary can be packaged) **+ the WS-1 embedding decision** (PyO3 compile-time embed vs uv-managed bundle — `DECISIONS.md:96`; WS-1 owns the pick per `02-ws2-distribution.md:64-73,193`). **Sequencing: the WS-1 embedding decision blocks size-gate finalization** — both on-disk options land ~30–40 MB; fetch-on-first-launch is disqualified by the offline requirement (`02-ws2-distribution.md:71-73`).
- **Scope:** Stand up the cross-compile CI for the 5 spec targets (macOS arm64/x86_64, Linux x86_64/arm64, Windows x86_64 — `specs/distribution.md:82`) on the existing self-hosted runners (native macOS for code-signing, native `esperie-linux-arm` to avoid QEMU; `02-ws2-distribution.md:30-37`). Apply the strip+prune size levers (release profile `strip="symbols"`, `panic="abort"`, `opt-level="z"`, `lto="fat"`, `codegen-units=1`; stdlib prune validated against an `envoy init` import trace, NOT guessed; `cargo bloat` on the Rust side; split-debuginfo to a sidecar artifact for the reproducible-build stream — `02-ws2-distribution.md:77-78`). The offline model is a SEPARATE artifact, never inside the size-gated binary (S17 owns it). Build under a codename + internal signing keys.
- **Acceptance criteria:**
  - **NOW (codename):** the codename binary builds green on each of the 5 targets on the self-hosted runners (per-arch macOS, NOT a `lipo` universal binary — universal doubles size against the ceiling, `02-ws2-distribution.md:34`).
  - **NOW (codename):** `tests/acceptance/phase_02/test_binary_size_under_50mb.py` is **<50 MB** (`du -sh`) on every one of the 5 targets; any target >50 MB fails CI (`specs/distribution.md:80-82`).
  - **NOW (codename):** debug symbols are stripped from the distributed binary; the debug-symbol bundle ships as a separate downloadable artifact (`specs/distribution.md:84`).
  - **NOW (codename):** the size measurement is taken on the produced static binary ONLY — no offline model counted against the gate (`specs/distribution.md:80`; `02-ws2-distribution.md:84-89`).
  - **AT RELEASE (legal-gated):** publishing the binary under the trademarked name on `get.envoy.ai`/brew/winget/cargo and attaching the composite LICENSE — held until ADR-0009 trademark `:312` + composite-LICENSE `:307` close. The reproducible-build attestation MUST be re-run on the production-named binary (name strings change the bytes; the codename attestation does NOT transfer — `02-ws2-distribution.md:174`).
- **Capacity check:** invariants ≈ 5 (per-target build, strip discipline, stdlib-prune-from-trace, size ceiling, model-is-separate); call-graph hops ≈ 3 (CI matrix → cargo profile → embed → `du` gate); LOC ≈ 400 load-bearing (CI config + Cargo profile + prune list + size-gate wiring; the 5 targets are pattern-stamped, not 5 independent invariants). Live feedback loop (`du -sh` + green build per cell, `autonomous-execution.md` § Feedback Loops eligible). **Within budget.** musl is deliberately NOT in this shard — it is the HIGH-risk cell, isolated in S15m.

## S15m — musl mostly-static + Alpine import-smoke gate (the HIGH-risk cell)

- **Type:** Build
- **Value-anchor:** `briefs/00-phase-02-scope.md` §WS-2 names the 5th target "Linux … musl"; EC-02.1 "Rust binary builds on 5 targets" (`01-analysis/00-objectives-to-deliverables.md:28`). ADR-0001 (`DECISIONS.md:98`) names musl in the cross-compilation matrix — the static-Linux target is what lets sovereignty-minded users run Envoy on Alpine/distroless with zero glibc dependency, the strongest single-binary story.
- **Implements:** `specs/distribution.md` §Phase 02 distribution (`:19-25`) musl/distroless cell; ADR-0001 musl matrix entry (`DECISIONS.md:98`). **Closes the spec gap** that musl is a 6th target named in ADR-0001 but absent from the size-gate's 5-target enumeration (`specs/distribution.md:82`; `02-ws2-distribution.md:26,183`) — flag for a `/todos`-time spec addition, do NOT edit the spec here (`rules/spec-accuracy.md`).
- **Depends:** **S15** (reuses S15's strip/prune/embed pipeline and size gate; musl is the one cell S15 deliberately excluded).
- **Scope:** Ship the musl target as a **mostly-static** binary — static-CRT-linked Rust + a `python-build-standalone` musl-built CPython that **retains the ability to load extensions** — NOT a 100%-static single ELF. **The load-bearing failure mode:** `dlopen` of C-extension `.so` files (the kailash-rs-bindings runtime is itself a compiled `.so`, ADR-0001 `:55`) breaks under full-static musl; `getaddrinfo`/NSS for the N=3 mirror DNS also misbehaves (`02-ws2-distribution.md:39-52`). Gate the cell behind an explicit **Tier-3 Alpine-container import-smoke test** that actually imports the kailash-rs-bindings `.so` and runs `envoy init` inside Alpine. **Fallback valve:** if the cell overruns its budget, the honest disposition is to ship `linux-gnu` only at Phase-02 and **defer musl to a follow-up with a tracking issue** — musl users run the gnu binary on glibc distros, and the pure-Python `kailash-py` escape (ADR-0001 `:75`) covers the rest (`02-ws2-distribution.md:54,194`). Pin the exact `python-build-standalone` release + mirror it so an upstream musl-variant drop does not strand the cell.
- **Acceptance criteria:**
  - **NOW (codename):** the Alpine import-smoke Tier-3 test passes — inside an Alpine container, the binary imports the kailash-rs-bindings `.so` AND completes `envoy init` (the exact extension-import failure mode unit tests cannot see; `02-ws2-distribution.md:49,51`).
  - **NOW (codename):** the musl binary passes the same `test_binary_size_under_50mb.py` <50 MB gate as S15's 5 targets (musl added as the 6th size-gate cell).
  - **NOW (codename):** N=3 mirror DNS resolution works from the musl binary (NSS/`getaddrinfo` under static musl verified, not assumed).
  - **Fallback gate (explicit valve):** IF the import-smoke gate cannot be made green within this shard's budget, ship gnu-only at Phase-02 + file a tracking issue with the musl deferral — this is an ACCEPTABLE disposition for THIS cell only (it is the named HIGH-risk fallback, `02-ws2-distribution.md:54`), NOT a precedent for the other 5 targets, which have no fallback.
  - **AT RELEASE (legal-gated):** same trademark/LICENSE/redistribution gates as S15 apply to the musl artifact when published.
- **Capacity check:** invariants ≈ 4 (mostly-static-not-full-static, extension-`dlopen`-survives, NSS/DNS works, import-smoke-is-the-gate); call-graph hops ≈ 3 (musl toolchain → python-build-standalone → `.so` import → Alpine harness); LOC ≈ 300 (CI cell + Alpine Tier-3 harness + pin/mirror config). Live loop (Alpine import-smoke fires in-session). **Within budget — and deliberately isolated so its HIGH risk cannot poison S15's size gate or the rest of M6** (`02-plans/01-architecture.md` WS-2 verdict; `journal/0006`).

## S16 — N=3 mirror + reproducible-build + key-rotation + install/upgrade/rollback surface + legal-gate release CI checks

- **Type:** Build+Wire
- **Value-anchor:** `briefs/00-phase-02-scope.md` §WS-2 "N=3 mirror layer + reproducible-build verification stream + signing-key rotation" (EC-02.9, `01-analysis/00-objectives-to-deliverables.md:36`). This is the supply-chain trust spine that makes "you can verify the binary you run was built from the source we published" a real, sovereignty-grade claim — the BET-3 sovereignty bet (`specs/distribution.md:11`).
- **Implements:** `specs/distribution.md` §N=3 mirror verification (`:27-33`), §Binary signing key rotation (`:35-37`), §Reproducible-build verification stream (`:39-41`), §Upgrade/Rollback/Uninstall (`:47-51`), §Jurisdictional advisories (`:53-57`), §Installer security (`:87-89`), §Error taxonomy (`:90-104`); `specs/threat-model.md` T-050a (mirror), T-050b (signing-key), T-060 (reproducible-build / binary poisoning).
- **Depends:** **S15** (the binary + its manifest hash are the trust objects the mirror layer, reproducible-build attestation, and revocation list all operate on).
- **Scope:** Build the signed-manifest trust object (`{binary_hash, version, signing_key_fingerprint, signature}` per target); installer fetches manifest from all 3 mirrors (Foundation GitHub / IPFS-pinned / community redistributor), verifies each signature against the pinned key, confirms `sha256(binary)` matches on **≥2 of 3** (`MirrorSignatureMismatchError`, Retry: Never; `specs/distribution.md:94`). Fetch over TLS 1.3 + pinned certs (`specs/network-security.md`). Land reproducible-build determinism per target (pinned `rustc`+CPython-standalone-hash+`Cargo.lock`, `--remap-path-prefix`, `SOURCE_DATE_EPOCH`, deterministic archive ordering; `ReproducibleBuildFailedError`, Retry: Never; `specs/distribution.md:95`) — attestation computed on the **pre-signature** binary (code-signing is non-deterministic; manifest records both unsigned-reproducible AND signed-distributable hash — flag the two-hash spec gap, `02-ws2-distribution.md:119,184`). Land old-key-co-signs-new-key rotation + always-fetched signed revocation list consulted BEFORE hash verification (`RevokedSigningKeyError`, Retry: Never; `specs/distribution.md:97`). Build `envoy upgrade` (hash-verify), rollback (30-day window, `RollbackWindowExpiredError`), `envoy uninstall --destroy-vault` (double-confirm), and the jurisdictional-advisory surface (`JurisdictionalGateRefusedError` enforcement logic — buildable now; only its jurisdiction-parameter LIST is export-gated, `02-ws2-distribution.md:175`). Wire the three legal-gate release CI checks as structural hard-fails.
- **Acceptance criteria:**
  - **NOW (codename, against test-fixture mirrors + throwaway test keys):** ≥2-of-3 mirror hash-match passes; a single divergent mirror is out-voted and install proceeds; all-3-divergent refuses with `MirrorSignatureMismatchError` (regression test, `specs/distribution.md:94`).
  - **NOW (codename):** reproducible-build determinism verified in-house — build twice per target, diff bytes equal on the pre-signature binary; a forced non-determinism triggers `ReproducibleBuildFailedError` (`specs/distribution.md:95`; T-060).
  - **NOW (codename):** revocation dominates hash-match — a binary whose `signing_key_fingerprint` is in the revocation list is refused even when its hash matches across mirrors (`RevokedSigningKeyError`; T-050b). Rotation chain validates a successor key from the original pinned key alone (no live key-server).
  - **NOW (codename):** `envoy upgrade`/rollback-30-day/`--destroy-vault`-double-confirm + each error-taxonomy entry (`specs/distribution.md:90-104`) behaves per spec; installer refuses install if the Trust Vault dir is world-readable (`TrustVaultDirWorldReadableError`).
  - **AT RELEASE (legal-gated):** the three release-time CI hard-fail checks fire — (1) composite-LICENSE present (`pip-licenses` diff, `DECISIONS.md:322`), (2) public name permitted (trademark cleared `:312`), (3) cross-border redistribution permitted for the target jurisdiction (export-control `:311`, the jurisdiction LIST is the gated parameter). Each is structural CI, not a human checklist (`02-ws2-distribution.md:172`).
- **Capacity check:** invariants ≈ 6 (≥2/3 mirror quorum, reproducible determinism per target, revocation-before-hash, rotation-chain-from-pinned-key, 30-day-rollback/destroy-vault, three-legal-gate-CI); call-graph hops ≈ 3–4 (mirror fetch → manifest verify → revocation check → hash confirm → install); LOC ≈ 480 load-bearing (manifest/quorum/revocation logic + upgrade/rollback/uninstall + release-gate CI; the error-taxonomy entries are pattern-stamped). Live loop (fixture-mirror + throwaway-key regression tests fire in-session). **At budget — the manifest+quorum+revocation+rotation composition is the load-bearing cluster; flag at reviewer time. If reproducible-build determinism across targets proves to exceed budget, split the determinism harness into a sibling shard rather than overflowing.**

## S17 — First-run onboarding + offline-model bundle / ADR-0006 degraded-mode + install-to-first-value <10min gate

- **Type:** Build+Wire
- **Value-anchor:** `briefs/00-phase-02-scope.md` §WS-2 "Install-to-first-value <10 min on mobile — MED-R5-1 acceptance gate" + "First-run offline-model bundle" (EC-02.10, `01-analysis/00-objectives-to-deliverables.md:37`; `distribution.md:59`). This is the user's literal first experience of Envoy — tap-install to the first Boundary-Conversation greeting in under 10 minutes, working even with zero network. It is what converts a downloaded binary into a _working agent the user can talk to_.
- **Implements:** `specs/distribution.md` §First-run flow (`:43-45`), §Install-to-first-value (`:59-68`), `OfflineFirstRunModelMissingError` (`:100`); ADR-0006 degraded-mode (`DECISIONS.md:210`, Consequences `:228` "minimal prompt-template runtime can drive Boundary Conversation without full LLM"). **Closes the spec gap** that offline-model-bundle size is unbounded relative to the 50 MB gate (`02-ws2-distribution.md:185`) — flag for `/todos`-time spec clarification, do NOT edit the spec.
- **Depends:** **S13 + S15.** S13 (WS-3 Flutter client + QR-pair) provides the mobile first-run surface the install-to-first-value clock measures; S15 provides the desktop binary the `curl|sh` + `envoy init` path measures.
- **Scope:** Build the first-run flow (runtime picker → model picker → Boundary Conversation → Shamir ritual → visible-secret setup, `specs/distribution.md:45`). Ship the **offline-model bundle as a SEPARATE artifact** — never inside the size-gated S15 binary (a local model is hundreds of MB to GB; `02-ws2-distribution.md:84-89`). Lean on the **ADR-0006 degraded-mode runtime** (minimal prompt-template runtime, no full LLM) as the TRUE offline zero-network path that drives the Boundary Conversation when the user declines cloud AND skips the local download (`DECISIONS.md:228`; `02-ws2-distribution.md:89`) — the bundled local model is then an optional larger download, resolving the offline-vs-size tension. Wire the install-to-first-value measurement substrate (mobile Flutter integration test recording install-complete → first-launch → S0-greet render; desktop `curl|sh` + `envoy init` + first Grant Moment render; `specs/distribution.md:63-68`).
- **Acceptance criteria:**
  - **NOW (codename):** the **install-to-first-value <10 min mobile / <5 min desktop** gate passes — median across 10 sample installs under each (`tests/acceptance/phase_02/test_install_to_first_value_mobile.py` + `_desktop.py`); Phase-02 exit is blocked if median exceeds the budget (`specs/distribution.md:61,68`). Clock starts at install-tap, stops at the first user-observable Envoy action (Boundary Conversation S0 greet).
  - **NOW (codename):** the **ADR-0006 degraded-mode zero-network path** drives the Boundary Conversation to S0 with NO network and NO downloaded model (decline-cloud + skip-local), proving the offline-first-run requirement (`specs/distribution.md:127`) without bloating the binary (`02-ws2-distribution.md:89`).
  - **NOW (codename):** the offline-model bundle is a distinct artifact, fetched/staged at first-run via the model picker, NOT embedded in the S15 binary; a missing/corrupt bundle surfaces `OfflineFirstRunModelMissingError` in degraded mode rather than crashing (`specs/distribution.md:100`).
  - **AT RELEASE (legal-gated):** the mobile first-run path publishes through App Store / Play Store under the trademarked name — held on the same trademark gate as S15/S16; the install-to-first-value _measurement_ runs NOW under codename, only the _public listing_ is gated.
- **Capacity check:** invariants ≈ 5 (separate-model-artifact, degraded-mode-is-the-offline-path, <10min/<5min measured gate, model-picker-fetch-not-embed, missing-model-degrades-not-crashes); call-graph hops ≈ 3 (first-run flow → model picker → degraded runtime → S0 render); LOC ≈ 400 (first-run flow + degraded-mode wiring + measurement harness; the picker UX is pattern, the timing gate is the load-bearing logic). Loop = medium (the mobile Flutter integration test + desktop shell-through harness fire in-session but cross the Flutter boundary). **Within budget. The offline-model-vs-50MB tension is the one subtle design invariant — call it out in the reviewer prompt: the bundle MUST be separate, degraded-mode MUST be the zero-network path.**
